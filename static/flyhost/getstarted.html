Getting Started

Recommended Hardware and OS

  • MacBook Pro (13-inch, 2020)

  • macOS v12.6 (21G115)

  • Min. 300MB disk space

Quickstart for MacOS

  1. Download the disk image file flyhost.dmg from Downloads

  2. Mount the disk image

  3. Drag and drop flyhost.app to Applications folder

  4. Allow the app permission to access Documents folder

  5. If the app loads successfully, you will be able to see the screenhot(s) in the sections below

Production Interface

  1. Drag and drop the folder containing tester output.stdf files into the Input Folder field. You may also choose to leave it as default path, then Open INPUT folder, then load the output.stdf files into the folder.

  2. Click STDF to CSV to start processing

  3. Click Open OUTPUT folder to open the result folders. The CSV test results files in our standard test datalog format will be located in the folder.

Engineering/Debugging Interface

  1. Use the file menu to open the folder containing configuration files

  2. Make copies of the config.yml file, rename as you like. Edit the configurations in the new copy

  3. Drag and drop the new copy of the .yml into the Drop recipe file field

  4. Drag and drop STDF file into the Drop STDF file field

  5. Read stdf first

  6. Export to Excel /or/ Evaluate the data (get all parametric FOMs) /or/ Evaluate the data (using user-configured recipe)
