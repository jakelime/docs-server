Creating recipes

A template recipe created by default out of the box. Use it as a guideline to make/create new recipes.

Create new Test Recipe

  1. Go to the Engineering tab

  2. Open the working config folder. You can use Open config folder from the File menubar.

  3. Make a New recipe file by duplicating config.yml.

  4. Edit the New_Recipe.yml using any text editor. Highly recommended to use good quality text editors such as SublimeText in the screenshot below (quality auto-formatting, and syntax highlighting).

  5. Refer to recipe section for configuration details.

  6. Save the modified recipe. Drag and drop into the UI. Load the recipe. Ensure that the recipe is loaded correctly by checking the recipe name and version.

Set test params

  1. Once test sequence (a.k.a. recipe) is configured properly, double check that there are no errors.

  2. Check that the data.tdms generated correctly as programmed. Use the read tdms function to convert to CSV if necessary.

  3. Drag and drop the data.tdms into the raw data field (it is also auto-updated after running test sequence). Click Eval to evaluate the raw data.

  4. If foms-PROJECT_NAME.csv file do not exist, the app will generate a template. Open the foms-XXXX.csv from the config folder to configure FOMs, data selection, function calls, test specs, correction factors and etc. This file will determine how the app will compute using the given data.tdms.
