Production usage

Guide for Operator usage for Production.

Basic checks

  1. Ensure that the correct recipe is loaded. (1) Load recipe using the File menu. (2) Use the UI to ensure that the correct recipe name and version is loaded.

    2 steps to check recipe

  2. Input in the lot name into the Lot name entry field. Illegal characters are not allowed (i.e. !@#$%^&*). Click on OPEN LOT.

  3. Load the DUT. Input SerialNumber entry field. Illegal characters are not allowed (i.e. !@#$%^&*). Click on START to trigger test.

  4. Example of a successful test

    1. Right hand side of the UI will display the current running lotname, and the last tested SerialNumber will be displayed.

    2. Lot run statistics (unique DUTs, total number of tests, number of unique DUTs that passed, number of unique DUTs that failed) will also be displayed on the RHS.

    3. To perform a retest, unload the DUT and load the DUT again. Instead of re-entering the same SerialNumber again, you can simply press the Retest button.

    4. Details on the PASS/FAIL test results will be displayed in the log. You can see which are the parameters that failed and the corresponding test specs.

    5. Test run time is also displayed in the log.

Viewing results

  1. Details on working directory (Open Local button)

  1. _configurations

    Contains recipes and parameteric definitions.

  2. 01-DataCSV

    Contains the output TestDatalog.csv upon end lot

  3. 02-WorkingDatalog_TDMS

    Vontains working data files in TDMS format during the lot running. In case of PC/OS/Software crash, you can resume back the lot by Open Lot with the same lot name. The app will perform smart checking using the existing working data files to resume the lot run.

  4. 03-WorkingDatalog_CSV

    Working with CSV file is an incredibly inefficient method. However, we recognise that it is an easy way for people to view the data. This folder will contain a snapshot of the working data file (TDMS) in CSV format. This folder will automatically be rolled over / cleared of old files.

  5. 04-Logfiles

    Each DUT will be logged individually. Visit the here to debug by DUT-test hardware level.

  6. 05-ArchiveDatalog

    All working data files in TDMS format will be automatically archived into this folder during End Lot. Nicely archived into folders by date and lotname
